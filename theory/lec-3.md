### Можно ли выразить анализ типов с предыдущей лекции как анализ над решетками?

Зависит от наличия типа `any`.
В реализации TypeAnalysis его не было, то есть следующий код не типизировался
```
f = 1;
f = alloc null;
```

Так как нет типов bottom и top, то решётку не сделать. Если их добавить, то можно использовать плоскую решетку

### Можно ли выразить анализ над решетками как анализ типов?

Можно сделать функцию f : L -> Types (Из элементов решётки в типы). В системе типов отношения нижней и верхней грани из решётки выражаются через подтипирование. Для верхней и нижней грани можно ввести типы `Top`/`Bottom`
