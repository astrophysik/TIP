### Какова сложность структурного алгоритма?

Пусть N - кол-во верщин CFG, k - кол-во переменных

Изначально в `WL` лежат все ноды CFG. Для каждой ноды:
1. Кол-во успешых `if x_i != y_i` <= N * K
1. Подсчёт значения `y = f_i(x_1, ... x_n)` - O(k)
2. `WL` <- `WL` \/ `deps(v_i)` - Может потенциально добавиться O(N) нод CFG

Итоговая сложность будет O(N^2 * k ^2)

### Если в CFG нет циклов?

При отсуствии циклов не нужно делать пересчёт (обновлять `WL`), сложность будет O(N * k)

### Какова сложность по памяти?

На каждом шаге можно хранить состояние для N нод размера k - итого O(N * k)

### Распишите и решите систему ограничений для very busy expressions анализа для примера

```
var x,a,b;          // {x > 0, x - 1, x - 2, a * b}
x = input;          // {x > 0, x - 1, x - 2, a * b}
a = x-1;            // {x > 0, x - 1, x - 2, a * b}
b = x-2;            // {x > 0, x -2, a * b}
while (x > 0) {     // {a * b}
    output a*b-x;   // {a * b - x, x - 1, x > 0, a * b}
    x = x-1;        // {x - 1, a * b, x > 0}
}
output a*b;         // {a * b}
```
