### Что будет, если в нашу систему ввести тип Bool?

1.

Изменившиеся правила
1. E1 == E2         | [E1] == [E2] && [E1 == E2] == Bool
2. if (E) {S}       | [E] == Bool
3. while (E) {S}    | [E] == Bool


---
2. Более полным анализ не станет, так как количество типизируемых программ не увеличится. 
Точность уменьшится, так как появится больше не типизируем программ

### Что будет, если в нашу систему ввести тип Array?

Синтаксическая конструкция Array<_Alpha_>

Новые правила
1. A = {E_1, ..., E_N} | forall i, j: [E_i] == [E_j] && [A] == Array<[E_1]>
2. A[I] | [I] == int && [A] == Array<_alpha_> && [ A[I] ] == _alpha_
3. A[I] op E | [I] == int && [A[I]] == [E] && [A] = Array<[E]>
4. {} | [{}] == Array<_alpha_>

Пример:
```
main() {
    var x, y, z, t;
    x = {2, 4, 8, 16, 32, 64};    // [[x]] == Array<Int>
    y = x[x[3]];                  // [[y]] == Int
    z = {{}, x};                  // [[z]] == Array<Array<Int>>
    t = z[1];                     // [[t]] == Array<Int>
    t[2] = y;
}
```

### Подумайте, что происходит в получившейся реализации, если в программе есть рекурсивный тип?

В реализации используется UnionFindSolver, который допускает использование рекурсивных типов

Пример:
```
main() {
    var p;
    p = alloc null;
    *p=p;
    return 0;
}
```
